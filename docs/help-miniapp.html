Solución de problemas
Si Iris no responde lo que esperás.
No encontró alertas activas
El sistema de monitoreo está limpio o hay un problema de conexión con PRTG. Si sabés que hay una alerta, revisá con tu administrador la conectividad a la instancia Banco/EKT.
"No se encontró template para esta IP"
La IP no tiene un template de aplicación configurado en la base de datos. Solicitá a tu administrador que asocie un template a esa IP para habilitar el escalamiento.
"Sin tickets históricos"
El equipo no tiene tickets registrados para ese sensor. Puede ser un equipo nuevo o que los incidentes anteriores fueron registrados bajo un sensor diferente. Probá sin especificar el sensor.
No tengo permisos para consultar alertas
Las tools de alertas (get_active_alerts, get_alert_detail) requieren permisos específicos en tu rol. Contactá a tu administrador para que los habilite.
La respuesta tarda mucho
get_alert_detail hace múltiples consultas en paralelo y puede tardar unos segundos. Si pasa de 15s, puede haber un problema de conectividad con PRTG o la base de datos.
Los contactos de escalamiento están vacíos
El inventario del equipo no tiene asignadas las áreas atendedora/administradora. Reportalo a tu administrador para que actualice el inventario en la base de datos.
No sé la IP del equipo
Primero preguntame qué alertas hay activas con "¿Qué alertas hay activas?". La respuesta incluye la IP de cada equipo afectado.
👥 Para problemas de acceso, permisos o comportamientos inesperados, contactá a tu administrador con la hora aproximada del incidente.
